HTML Attributes Configuration

Attribute Type Default Description Example
items-per-page Number 3 Number of items to display per page. Must be a positive integer. items-per-page="5"
prev-text String "Prev" Text or HTML content for the previous button. prev-text="‚Üê Back"
next-text String "Next" Text or HTML content for the next button. next-text="Forward ‚Üí"
id String auto Unique identifier for programmatic access via JavaScript. id="my-pagination"
Pro Tips
  • You can use HTML in prev-text and next-text for icons: prev-text="<i class='fas fa-arrow-left'></i> Back"
  • The items-per-page attribute can be changed dynamically and the component will automatically recalculate
  • Always provide meaningful text for accessibility, even when using icons

JavaScript Methods & Properties

Navigation Methods
goToPage(pageNumber)

Navigate to a specific page number (1-based index).

                                                        
                                                            const pagination = document.getElementById('my-pagination');
pagination.goToPage(3); // Go to page 3
refresh()

Recalculate pagination after adding/removing items.

                                                        
                                                            // After adding new content
pagination.appendChild(newItem);
pagination.refresh();
Getter Methods
getCurrentPage()

Returns the current active page number.

                                                        
                                                            const currentPage = pagination.getCurrentPage();
console.log(`Current page: ${currentPage}`);
getTotalPages()

Returns the total number of pages available.

                                                        
                                                            const totalPages = pagination.getTotalPages();
console.log(`Total pages: ${totalPages}`);
Important Notes
  • Always call refresh() after dynamically modifying the slotted content
  • Page numbers are 1-based, not 0-based (first page is page 1)
  • Methods will validate input and ignore invalid page numbers
  • All methods are safe to call at any time, even before the component is fully initialized

Custom Events

pagechange Event

Fired whenever the user navigates to a different page, either through UI interaction or programmatic navigation.

Event Detail Properties:
Property Type Description
currentPage Number The new current page number (1-based)
totalPages Number Total number of pages available
itemsPerPage Number Number of items displayed per page
startIndex Number 0-based index of the first item on current page
endIndex Number 0-based index of the last item on current page
Event Listener Examples:
                                                
                                                    // Listen to a specific pagination component
document.getElementById('my-pagination').addEventListener('pagechange', function(event) {
    console.log('Page changed to:', event.detail.currentPage);
    console.log('Items on this page:', event.detail.startIndex, 'to', event.detail.endIndex);
    
    // Update URL or analytics
    updateURL(event.detail.currentPage);
    trackPageView(event.detail);
});

// Listen to all pagination components on the page
document.addEventListener('pagechange', function(event) {
    console.log('Any pagination changed:', event.detail);
    
    // Update global state or perform common actions
    updateGlobalState(event.detail);
});
Event Best Practices
  • Use pagechange events to update URL fragments or query parameters
  • Events bubble up, so you can listen at the document level for all pagination components
  • The event is fired after the UI has been updated with new content
  • Perfect for analytics tracking, lazy loading, or updating related components

3. Advanced JavaScript Control 
                                                
                                                    <script>
// Wait for the component to be ready
document.addEventListener('DOMContentLoaded', function() {
    const pagination = document.getElementById('my-pagination');
    
    // Control buttons
    document.getElementById('first-page').addEventListener('click', () => {
        pagination.goToPage(1);
    });
    
    document.getElementById('last-page').addEventListener('click', () => {
        pagination.goToPage(pagination.getTotalPages());
    });
    
    // Add new items dynamically
    document.getElementById('add-item').addEventListener('click', () => {
        const newItem = document.createElement('div');
        newItem.slot = 'items';
        newItem.innerHTML = `<h4>New Item ${Date.now()}</h4>`;
        
        pagination.appendChild(newItem);
        pagination.refresh(); // Important: refresh after adding items
    });
    
    // Listen for page changes
    pagination.addEventListener('pagechange', (event) => {
        console.log(`Navigated to page ${event.detail.currentPage}`);
        
        // Update URL without page reload
        const url = new URL(window.location);
        url.searchParams.set('page', event.detail.currentPage);
        history.replaceState(null, '', url);
        
        // Load data for the new page (if using AJAX)
        loadPageData(event.detail.currentPage);
    });
});

// Function to load page data (example)
function loadPageData(page) {
    // Your AJAX logic here
    fetch(`/api/data?page=${page}`)
        .then(response => response.json())
        .then(data => {
            // Update the pagination items with new data
            updatePaginationItems(data);
        });
}
</script>

5. Framework Integration
React Integration:
                                                
                                                    import React, { useRef, useEffect } from 'react';

function ProductList({ products }) {
    const paginationRef = useRef();
    
    useEffect(() => {
        const handlePageChange = (event) => {
            console.log('Page changed:', event.detail.currentPage);
        };
        
        const pagination = paginationRef.current;
        pagination.addEventListener('pagechange', handlePageChange);
        
        return () => {
            pagination.removeEventListener('pagechange', handlePageChange);
        };
    }, []);
    
    return (
        <pagination-wc ref={paginationRef} items-per-page="6">
            {products.map(product => (
                <div key={product.id} slot="items">
                    <ProductCard product={product} />
                </div>
            ))}
        </pagination-wc>
    );
}
Vue.js Integration:
                                                
                                                    <template>
    <pagination-wc 
        :items-per-page="itemsPerPage"
        :prev-text="prevText"
        :next-text="nextText"
        @pagechange="onPageChange"
        ref="pagination">
        
        <div 
            v-for="item in items" 
            :key="item.id" 
            slot="items">
            {{ item.content }}
        </div>
    </pagination-wc>
</template>

<script>
export default {
    data() {
        return {
            itemsPerPage: 5,
            prevText: '‚Üê Back',
            nextText: 'Next ‚Üí',
            items: []
        };
    },
    methods: {
        onPageChange(event) {
            console.log('Vue page change:', event.detail);
            this.$emit('page-changed', event.detail);
        },
        goToFirstPage() {
            this.$refs.pagination.goToPage(1);
        }
    }
};
</script>
